# 客户端使用

## 启动参数

示例：

```bash
CfSpeedtest.Client.exe --server https://example.com --client-id 117422062b8641f997175a1aefb3501c --isp Unicom --name GX-Cu --auto-update
```

## 心跳与在线状态

客户端会按服务端下发的心跳间隔持续发送心跳。

作用：

- 更新在线状态
- 保持最后活跃时间
- 等待统一轮次开始时仍显示为在线

## 统一轮次测速

客户端拿到任务后不会立刻开始，而是等待服务端给定的统一开始时间。

测速过程中：

- 先做 TCP 延迟 / 丢包测试
- 再做下载测速
- 若达标数量不足，会继续向服务端补拉新 IP
- 重复 IP 不会重复测速

## 速度阈值与限速

服务端可配置：

- 最低下载速度阈值
- 最高下载速度限速

结果逻辑：

- 低于最低阈值的结果不进入最终 TopN
- 仅最终达标结果会上报和参与 DNS 聚合
