# 白名单与客户端管理

## 严格白名单模式

启用后：

- 未预生成的 clientId 不允许注册
- 未在允许列表中的 clientId 不允许心跳、拉任务、上报

## 推荐流程

1. 在“客户端”页面开启严格白名单模式
2. 使用“客户端启动参数生成”区域生成新的 clientId
3. 复制完整启动命令到目标设备执行

## 单个客户端控制

客户端列表支持：

- 允许 / 禁用连接
- 删除客户端

## clientId 的使用方式

clientId 可以：

- 由服务端预先生成并加入允许列表
- 客户端启动时通过 `--client-id` 显式传入
- 客户端本地保存在 `client-state.json` 中以便后续复用
